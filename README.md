## 9. Висновки

### 📊 Порівняння моделей
У проєкті було протестовано три моделі: **Logistic Regression**, **Decision Tree** та **Random Forest**.

* **Random Forest** показав найвищу загальну точність ($Accuracy \approx 0.79$).
* **Logistic Regression** продемонструвала найкращий баланс за метрикою **ROC-AUC (~0.83)** та найвищий **Recall (0.79)** для класу відтоку.
* **Decision Tree** показала середні результати та поступається іншим моделям.

👉 Висновок: вибір моделі залежить від задачі — якщо важливо знаходити клієнтів, які підуть, кращою є **Logistic Regression**, якщо важлива загальна точність — **Random Forest**.

---

### ⚖️ Компроміс Precision vs Recall

* **Logistic Regression:**
  * Recall (Churn=1): **0.79**
  * Precision: **0.50**

* **Random Forest:**
  * Recall: **0.48**
  * Precision: **0.64**

👉 Logistic Regression знаходить більшість клієнтів, які підуть, але має більше хибних спрацювань.  
👉 Random Forest є точнішим у передбаченнях, але пропускає значну частину клієнтів із відтоком.

---

### 📉 Особливості даних

* Частка клієнтів без відтоку: **~73%**
* Частка клієнтів з відтоком: **~27%**

👉 Дані є **незбалансованими**, що впливає на якість моделей.  
Використання параметра `class_weight='balanced'` у Logistic Regression є обґрунтованим рішенням.

---

### 🔍 Ключові фактори впливу

На основі аналізу важливості ознак (**Random Forest**) та EDA:

1. **Tenure:** клієнти з малим терміном користування частіше припиняють користування.
2. **MonthlyCharges / TotalCharges:** вищі витрати пов’язані з більшою ймовірністю відтоку.
3. **Contract:** клієнти з типом **Month-to-month** мають більший ризик відтоку.

---

### 📌 Загальний висновок

* Моделі показали хорошу здатність до передбачення відтоку (**ROC-AUC ~0.83**).
* Навіть проста модель (**Logistic Regression**) може ефективно вирішувати задачу churn prediction.
* Вибір моделі залежить від бізнес-пріоритетів:
  * максимальне виявлення churn → Logistic Regression
  * точність передбачень → Random Forest
